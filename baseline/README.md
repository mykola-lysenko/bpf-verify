# baseline/

`results.json` is the committed per-target baseline that CI compares each
run against (`scripts/check_results.py`). It records, for every target, the
expected verifier verdict and instruction/state counts, so CI catches:

- targets that disappear from the results (moved/renamed kernel sources),
- verdict regressions (success → failure/skip),
- instruction-count collapse (a harness that got constant-folded and no
  longer exercises the target code),
- new targets added without a baseline entry.

## Bootstrapping / refreshing

The baseline must be generated by a real pipeline run (CI or a machine with
the kernel tree and uml-veristat set up):

```bash
python3 pipeline.py
python3 scripts/check_results.py output2/results.json --update-baseline
git add baseline/results.json && git commit
```

Until `results.json` exists here, CI reports a warning and skips the
comparison. After the first green run, download the `veristat-results`
artifact and commit its `results.json` via `--update-baseline`, or run the
commands above locally.

Non-fatal instruction-count drift is printed in the CI log; refresh the
baseline when drift is expected (kernel or LLVM version bumps).
